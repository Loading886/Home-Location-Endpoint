这是什么？

一套开源安装器 + 运行时，把一台干净的 Linux 落地机部署成「Apple 网络定位修改端点」。

iPhone 在查询周边 Wi-Fi / 蜂窝定位时会向 Apple 的 gs-loc.apple.com 等域名发请求。本项目在落地机上终止这些请求的内层 TLS，把返回结果里的经纬度平移到「落地机出口 IP 所在城市」内的一个随机坐标，再交还给手机——于是依赖网络定位的 App 会认为你在那座城市。

💡

关键点：手机不需要越狱、不需要装任何 App。只要信任一张本机生成的私有 CA，并让流量走全局隧道即可。普通上网流量不受影响、不经过拦截器。

工作原理

四步闭环，作用域被刻意收窄到 Apple 定位域名。

手机全隧道连接落地机

客户端使用安装器生成的传输协议连接落地机，开启全局 TUN/VPN，让系统级流量（含定位服务）都进隧道。

落地机识别定位请求

只有 Apple 网络定位请求会进入定位修改器，普通网页、App 和其它网络流量仍从落地机正常访问互联网。

把定位移到目标城市

定位修改器把结果调整到落地机出口 IP 所在城市内的随机点，再把结果返回手机。

保持自然稳定

定位会在设定点附近做轻微、平滑的变化；需要换一个点时可以重新随机选择，无需重装手机配置。

为什么用它

工程上尽量克制、可审计、可回滚。

🎯

作用域极窄

只路由 Apple 定位域名到定位修改器；证书与分流规则层层收窄，普通流量与其它站点不受影响。

🗺️

真实城市随机点

按出口 IP 识别城市，在 OpenStreetMap 行政边界内随机取点；取不到边界时在 IP 中心 3km 内回退并明确标记。

🔐

私钥即用即删

每台落地机生成独立 CA，只签一张叶证书后立即删除 CA 私钥；安装输出 CA 的 SHA-256 供你核对。

↩️

事务化安装

安装/升级用互斥锁、预检查与文件级事务；服务或完整性检查失败会恢复到先前受管状态。

⏸️

随时暂停

hle pause 暂停坐标改写并返回 Apple 原始定位；hle resume 即时恢复，代理与普通流量不中断。

🧩

传输参数校验

由安装器统一生成并现场校验所选协议需要的连接参数，避免节点配置与服务器状态不一致。

📲

自动交付描述文件

交互式安装完成后自动启动一次性下载链接和二维码；成功下载一次或 100 分钟后自动关闭。

🛡️

不动 SSH / 防火墙

安装器不改 SSH 端口/密钥/密码，也不会主动启用原本关闭的 UFW。

🤖

手机菜单控制

进阶模式可在 Telegram 菜单里切换、增加或删除地点，恢复真实定位，并直接获取节点链接和 CA 描述文件。

支持的系统与前提

项目要求
操作系统Debian 12 / 13，Ubuntu 22.04 / 24.04（systemd 作 PID 1）
CPU 架构完整/进阶模式 amd64 / arm64；仅定位模式不限架构
内存建议 ≥ 384 MiB（低于会警告）
磁盘完整模式 ≥ 200 MiB，仅定位模式 ≥ 50 MiB 根分区可用
网络出口能访问 GitHub、IP 定位服务、Nominatim、所选传输协议依赖的目标与 Apple 源站
客户端兼容安装器生成的节点协议，并能开启全局 TUN/VPN
✅

没有落地机？任何一台干净的上述系统 VPS 都可以。建议用一台专门的机器，不要装到还承载其它代理服务的主机上（完整模式会拒绝）。

安全与局限（请务必读）

本项目会扩大设备信任面，且并不能改写所有定位信号。诚实地了解边界，再决定是否使用。

⛔

不要依赖本项目处理紧急呼叫、Find My、防盗、合规或任何人身安全场景。请只在你拥有、且获得授权的设备与服务器上，用于学习、测试与隐私自保。

  • 手机原始的 Apple 定位请求仍会转发给 Apple，Apple 仍可能从 Wi-Fi BSSID、蜂窝信息或账号推断真实位置。
  • 只覆盖 Apple 网络定位。不保证覆盖 GPS/GNSS、蓝牙、蜂窝基带、紧急定位、证书固定或 App 自有定位；户外 GPS 会盖过网络定位。
  • 安装私有 CA 会扩大信任面。项目已删除 CA 私钥、并把路由与叶证书限定到 Apple 定位域名，但你仍需理解并接受该风险。
  • 节点和 Telegram 控制器按单人使用设计，不要把同一套节点凭据分发给互不信任的用户。
  • 若客户端没有把 UDP、IPv6、硬编码 IP 或系统服务流量纳入全隧道，服务端无法补救这种绕过。
  • 随机点可能落在城市内的公园、水域或工业区等不适合模拟日常活动的位置。
