用聊天菜单切换定位

项目继续替你管理代理、证书与定位路由，同时加入一个低权限 Telegram Bot。切换地点、增加或删除地点、暂停改写，都不需要 SSH，也不用重启代理连接。

📍

即时切换

点击城市按钮后立即生效，并自动恢复定位改写。

➕

自管地点

菜单内新增、确认、删除地点，最多保存 50 个。

🌍

还原定位

暂停坐标改写，代理与 Apple 请求转发保持在线。

安装前准备

  1. 在 Telegram 打开 @BotFather，创建一个只给本节点使用的新 Bot。
  2. 向新 Bot 发送一次 /start。
  3. 准备 Bot Token 和你自己的数字 Chat ID。不要复用正在被其他程序轮询的 Bot。
🔒

本项目使用 Telegram 官方 HTTPS 长轮询，不配置 webhook，也不开放 Bot 入站端口；只有安装时指定的 Chat ID 会被处理。

一键安装进阶模式

运行安装器，选择 2，再选择 VLESS + REALITY（推荐）或 SS2022。Token 输入不会回显，写入前会先校验 Bot 和 Chat。

curl -fsSL https://raw.githubusercontent.com/Loading886/Home-Location-Endpoint/main/install.sh \
  | sudo bash

无人值守安装示例：

curl -fsSL https://raw.githubusercontent.com/Loading886/Home-Location-Endpoint/main/install.sh \
  | sudo env \
      HLE_TELEGRAM_BOT_TOKEN='BOT_TOKEN' \
      HLE_TELEGRAM_CHAT_ID='NUMERIC_CHAT_ID' \
      bash -s -- --mode advanced --protocol vless-reality --port 443
⚠️

安装器不支持原地转换模式或协议。已经用 full / modifier-only 安装的机器，不要直接用 advanced 覆盖；需要先按文档安全卸载，再重新安装并更新客户端节点。

菜单怎么用

向 Bot 发送 /menu、/start、/location 或 /status 都会打开定位菜单。

操作行为
点击城市切换为该地点并恢复改写；无需重启服务或重连代理
🌍 真实定位暂停响应改写；代理和 Apple 请求转发仍保持运行
➕ 增加地点依次输入短名称、地址和 WGS84 格式的“纬度, 经度”，确认后保存
➖ 删除地点二次确认后删除非活动地点；活动地点和最后一个地点不能删除

首次安装会生成出口 IP 城市随机点，并预置洛杉矶、东京、香港、新加坡、吉隆坡、巴黎、法兰克福、Reykjavík 与南极昆仑站。每台服务器的随机点不同，重复安装会保留现有地点库。

南极等高纬地点

昆仑站会在安装时于站点附近生成独立随机中心，但关闭每 120 秒的平滑微漂移，避免高纬稀疏覆盖区反复无定位。v0.2.2 对两类 Apple 响应增加了有界恢复：

  • 纯 no-fix sentinel 批次生成目标点 45 米内的稳定微型坐标簇；稀疏批次只用同一进程近期真实出现的 Wi-Fi 身份补足。
  • WifiTile 404 依次尝试内存完整瓦片、Apple 公共种子和近期真实身份最小瓦片；没有可信模板就保留原始 404。
🧠

这些缓存只存在内存，带 TTL 和数量上限，不写入磁盘或普通日志；服务重启后自动清空。

权限与健康检查

Bot 运行在独立的 home-location-bot 低权限账号，不能读取节点 URI、REALITY/SS 凭据、CA 叶私钥或 install.env。地点更新采用原子替换，修改前保留最近 30 份本地备份。

sudo hle status
sudo hle verify
sudo systemctl status home-location-telegram-bot --no-pager
sudo journalctl -u home-location-telegram-bot --since '30 minutes ago' --no-pager

hle verify 会检查 Bot 最近 180 秒的 API 心跳。Token 被撤销、网络无法访问 Telegram，或同一 Token 被另一个控制器占用时，检查会明确失败。

⛔

近期 Wi-Fi 身份缓存以单个拦截器进程为边界。不要把同一节点凭据分发给互不信任的用户。

协议选择

🛡️

VLESS + REALITY + Vision

推荐公网新部署；固定 SNI/target 为 www.usc.edu:443，适合前面加 Realm 纯 TCP 中转。

🔁

SS2022

兼容选项，同端口需要 TCP 与 UDP；Realm 只转发 TCP，不能承载 SS2022 原生 UDP。Xray 已提示该实现未来可能弃用。

➡️

需要 Realm 前置、NAT 双 IP 或客户端链式代理？继续查看 前置配置指引。
