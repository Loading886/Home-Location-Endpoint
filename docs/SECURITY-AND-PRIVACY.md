# 安全与隐私

## 信任边界

手机对本项目生成的私有 CA 开启完全信任。即使 Xray 只把有限域名送进拦截器，CA 仍属于高权限
凭据。安装前应核对 CA SHA-256，服务器不再受信任时应从所有设备删除描述文件。

降低风险的实现：

- 每台落地机生成独立 CA，不复用仓库或作者的证书。
- CA 私钥仅用于签发一张叶证书，随后删除。
- 叶证书 SAN 只覆盖两个 `gs-loc` 域名和 `*.ls.apple.com`。
- Xray 路由进一步收窄到 `gs-loc` 与 `gspe85` 定位域名。
- 同一域名集合的 QUIC 被阻断以促使 TCP 回退，避免服务端已收到的 UDP 定位流量直接旁路。
- 拦截器只监听回环，systemd 使用独立低权限用户和文件系统限制。
- 公共版本没有原始请求/响应抓包功能，也不记录坐标或 URL 查询参数。
- HTTP 解析拒绝重复 framing header、CL/TE 歧义、折叠 header、控制字符和超限 body；并限制
  并发 worker、压缩后大小和日志文件大小。
- 只有解码器证明整个可识别批次都是 Apple no-fix sentinel 时才生成 45 m 微型簇；未知、畸形
  或无法建立安全锚点的响应仍按默认策略失败关闭，不会静默回传真实坐标。

## 仍然存在的风险

- 叶私钥位于落地机。服务器被控制后，攻击者可冒充该叶证书 SAN 覆盖的域名。
- Apple 原始定位请求仍由落地机转发给 Apple，可能包含真实环境标识。
- REALITY 认证失败的连接会被转发到伪装 target；target 选取不当可能形成受限的第三方转发。
- 安装器会拒绝解析到私网/保留地址的 REALITY target，并为 fallback 生成随机双向限速，但限速
  以单连接为单位，无法彻底消除大量并发连接滥用；它也可能形成可观察特征。
- 完整模式固定使用 `www.usc.edu:443`，并实时校验证书链、主机名、TLS 1.3 和 HTTP/2；这只能
  证明安装当时满足条件，不能保证该站点长期稳定或愿意承载 REALITY fallback 流量。
- IP 定位与城市边界服务会看到落地机 IP 和一次城市查询。
- `hle pause` 会让 Apple 原始定位响应返回手机；这意味着暂停期间设备可能重新显示真实位置，
  但代理和限定 TLS 拦截仍保持运行。恢复前应确认这是操作者希望的行为。
- `hle profile serve` 是明确启动、单次下载、最长 100 分钟的临时 HTTP 服务。随机令牌可防止
  普通端口扫描直接取得描述文件，但不能抵御链路上的主动替换；必须通过 SSH 终端独立核对
  CA SHA-256，或改用 SCP/SFTP/可信 HTTPS。描述文件只含 CA 公钥证书，不含 CA 私钥。
- 系统、Xray、Python 或 OpenSSL 的漏洞可能突破预期边界。

## 日志

默认只记录：请求类别、域名、无查询参数的路径、状态码、响应大小、是否改写、错误摘要。
默认不记录原始请求体、响应体、BSSID、坐标、账号标识或 VLESS URI。单个活动日志文件硬限制
为 16 MiB，logrotate 每日检查、最多保留 14 份并压缩；达到硬限制后文件日志暂停写入，systemd
journal 仍按主机自身策略记录运行消息。

## 防火墙

完整模式只在检测到 UFW 已经启用时添加 VLESS TCP 端口，不会启用 UFW，也不会修改 SSH 规则。
仅定位模式完全不修改 UFW。
云厂商安全组需要操作者自行设置。若使用 Realm 前置机，落地端口应只允许前置机来源。
临时 profile 下载端口也不会被自动放行；只应在下载期间开放，并在命令退出后撤销相应云防火墙、
NAT、Realm 或 UFW 规则。

## 漏洞报告

请不要在公开 Issue 中粘贴节点 URI、UUID、REALITY 密钥、CA/叶私钥、服务器 IP、日志原文或
抓包。报告流程见仓库根目录的 `SECURITY.md`。
