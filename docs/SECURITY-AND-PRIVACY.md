# 安全与隐私

## 信任边界

手机对本项目生成的私有 CA 开启完全信任。即使 Xray 只把有限域名送进拦截器，CA 仍属于高权限
凭据。安装前应核对 CA SHA-256，服务器不再受信任时应从所有设备删除描述文件。

降低风险的实现：

- 每台落地机生成独立 CA，不复用仓库或作者的证书。
- CA 私钥仅用于签发一张叶证书，随后删除。
- 叶证书 SAN 只覆盖两个 `gs-loc` 域名和 `*.ls.apple.com`。
- Xray 路由进一步收窄到 `gs-loc` 与 `gspe85` 定位域名。
- 拦截器只监听回环，systemd 使用独立低权限用户和文件系统限制。
- 公共版本没有原始请求/响应抓包功能，也不记录坐标或 URL 查询参数。

## 仍然存在的风险

- 叶私钥位于落地机。服务器被控制后，攻击者可冒充该叶证书 SAN 覆盖的域名。
- Apple 原始定位请求仍由落地机转发给 Apple，可能包含真实环境标识。
- REALITY 认证失败的连接会被转发到伪装 target；target 选取不当可能形成受限的第三方转发。
- IP 定位与城市边界服务会看到落地机 IP 和一次城市查询。
- 系统、Xray、Python 或 OpenSSL 的漏洞可能突破预期边界。

## 日志

默认只记录：请求类别、域名、无查询参数的路径、状态码、响应大小、是否改写、错误摘要。
默认不记录原始请求体、响应体、BSSID、坐标、账号标识或 VLESS URI。日志保留八周并压缩轮转。

## 防火墙

安装器只在检测到 UFW 已经启用时添加 VLESS TCP 端口，不会启用 UFW，也不会修改 SSH 规则。
云厂商安全组需要操作者自行设置。若使用 Realm 前置机，落地端口应只允许前置机来源。

## 漏洞报告

请不要在公开 Issue 中粘贴节点 URI、UUID、REALITY 密钥、CA/叶私钥、服务器 IP、日志原文或
抓包。报告流程见仓库根目录的 `SECURITY.md`。
